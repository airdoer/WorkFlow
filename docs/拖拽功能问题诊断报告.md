# 拖拽功能问题诊断报告

## 问题描述

在画布上点击节点或拖拽创建节点后，画布上没有相关节点显示。同时观察到 `gedit-playground-pipeline` 的 `top` 和 `height` 值持续增大（如 `top: 26065.5px, height: 51338px`），即使不做任何操作也在无限膨胀。

---

## 问题一：节点不可见

### 根因

**`FreeLayoutEditorProvider` 缺少 `materials` prop，导致画布节点渲染器为空。**

### 渲染链路

```
FreeLayoutEditorProvider
  → createDefaultPreset()
    → createMaterialsPlugin(opts.materials || {})
      → 注册 "node-render" 渲染器 = opts.renderDefaultNode || (() => null)  ← 问题所在
  → FlowNodesContentLayer
    → getPortalRenderer()
      → 查找 meta.renderKey || "node-render" 对应的 React 组件
      → 用该组件渲染每个节点
```

### 关键代码（@flowgram.ai/materials-plugin/dist/esm/index.js）

```js
var createMaterialsPlugin = definePluginCreator({
  onInit(ctx, opts) {
    const registry = ctx.get(FlowRendererRegistry);
    registry.registerReactComponent(
      FlowRendererKey.NODE_RENDER,
      opts.renderDefaultNode || (() => null)  // 没传则默认 () => null
    );
  }
});
```

没传 `materials.renderDefaultNode` → 所有节点渲染为 `null` → 画布上看不到节点。

### 与 formMeta.render 的区别

各节点 registry 中定义的 `formMeta.render` 是**节点内部表单内容**的渲染器，由 Node Engine 在 `NodeRender` 组件中调用。但它需要先有一个画布级的**节点外壳组件**（即 `renderDefaultNode`）来承载它。缺少外壳组件，`formMeta.render` 的内容也不会被渲染。

### 修复

添加 `materials.renderDefaultNode`，使用 `WorkflowNodeRenderer`（节点外壳）+ `NodeRender`（表单内容）：

```tsx
import { WorkflowNodeRenderer } from '@flowgram.ai/free-layout-editor';
import { NodeRender } from '@flowgram.ai/form-core';

materials={{
  renderDefaultNode: ({ node }) => (
    <WorkflowNodeRenderer node={node} style={{ background: '#fff', border: '1px solid #d9d9d9', borderRadius: 8, padding: 12, minWidth: 180 }}>
      <div style={{ fontWeight: 600, fontSize: 13, marginBottom: 8 }}>
        {node.getNodeMeta?.()?.title || node.type}
      </div>
      <NodeRender node={node} />
    </WorkflowNodeRenderer>
  ),
}}
```

## 问题二：画布无限膨胀（top/height 持续增大）

### 根因

**`.gedit-playground-container` 没有确定的高度约束，导致 FlowGram 的 `resize()` 形成正反馈循环。**

### 详细分析

#### DOM 结构

```
<div style="flex:1; position:relative">            ← 外层定位容器
  <div class="gedit-playground-container"           ← EditorRenderer 创建
       style="width:100%; height:100%">             ← 原始设置，不可靠
    <div class="gedit-playground"                   ← absolute 定位
         style="position:absolute; width:100%; height:100%">
      <div class="gedit-playground-pipeline" ...>   ← 画布内容
    </div>
  </div>
</div>
```

#### 反馈循环机制

FlowGram 的 `resize()` 方法（`@flowgram.ai/core/dist/esm/index.js`）：

```js
resize(msg, scrollToCenter = true) {
  if (!msg) {
    const boundingRect = this.node.getBoundingClientRect();
    msg = { width: boundingRect.width, height: boundingRect.height };
  }
  // ...
  if (scrollToCenter && height && Math.round(msg.height) !== height) {
    scrollY += (height - msg.height) / 2;  // ← 每次高度变化都会调整 scrollY
  }
  this.config.updateConfig({ ...msg, scrollX, scrollY });
}
```

循环过程：

1. `resize()` 读取 `getBoundingClientRect()` → 得到一个较大的 height（因为容器没有被约束）
2. 更新 config → `scrollY` 调整 → pipeline 的 `top` 改变
3. pipeline 的 `top` 改变 → 影响 `.gedit-playground` 的可滚动区域 → 影响 `getBoundingClientRect`
4. 触发新的 resize → 再次调整 `scrollY` → `top` 和 `height` 无限增长

#### 核心问题

- `.gedit-playground-container` 使用 `width:100%; height:100%`，但 `.gedit-playground` 是 absolute 定位不撑开容器
- 容器没有 CSS 类约束，`100%` 高度依赖父元素，而父元素的 flex 高度可能不稳定
- `gedit-playground-container` 没有任何 CSS 规则（FlowGram 未提供），完全依赖 style prop

### 修复

**使用 `position: absolute` + 四向偏移替代 `width/height: 100%`**，确保容器尺寸精确匹配其定位参考：

```tsx
<div style={{ flex: 1, position: 'relative', minHeight: 0, overflow: 'hidden' }}>
  <FreeLayoutEditorProvider ...>
    <EditorRenderer style={{ position: 'absolute', top: 0, left: 0, right: 0, bottom: 0 }} />
  </FreeLayoutEditorProvider>
  <PropertyPanel ... />
</div>
```

关键点：
- 外层 div 有 `position: relative` 作为定位参考
- `EditorRenderer` 使用 absolute 定位精确填满父容器
- `minHeight: 0` 防止 flex item 默认最小高度导致的膨胀
- `overflow: hidden` 阻止内容溢出

## 验证方式

1. 启动前端开发服务器
2. 进入 Workflow 页面
3. 打开 DevTools → Elements，检查 `.gedit-playground-pipeline` 的 `top` 和 `height` 应为合理值（如 `top: 0, height: ~800px`），且不再持续增大
4. 点击左侧 Toolbox 中的节点类型
5. 确认画布上出现可见的节点（带标题、表单字段、连接端口）
6. 确认节点可拖拽移动
7. 确认点击节点可选中
8. 确认右侧 PropertyPanel 显示对应节点的属性配置
